Sistema de Reubicación de Usuarios Afectados por el Cierre de Gimnasios

1. Descripción del problema

Durante los últimos años, muchas personas han contratado membresías en gimnasios con el objetivo de mejorar su salud y condición física. Sin embargo, el cierre repentino de centros deportivos puede dejar a los usuarios sin acceso al servicio contratado y sin información clara sobre alternativas disponibles. Por esto, el proyecto propone un sistema que permita buscar, comparar y recomendar gimnasios alternativos según las necesidades del usuario.

2. Análisis del problema

El problema se sitúa en el contexto de usuarios que pierden el acceso a su gimnasio por cierre de una sede o cadena deportiva. Los principales elementos que participan son el usuario afectado, los gimnasios disponibles, las membresías ofrecidas, los servicios de cada gimnasio y la solicitud de reubicación. El sistema interactúa con el usuario mediante una interfaz gráfica donde se ingresan sus preferencias y se muestran recomendaciones ordenadas por compatibilidad. En la versión actual se cargaron gimnasios y sedes de Santiago y comunas cercanas, usando valores referenciales para poder probar la comparación de precios, membresías y servicios.

3. Definición de requerimientos

Caso de uso 1: Buscar recomendación de gimnasio

El usuario ingresa su nombre, comuna, presupuesto máximo, tipo de membresía y servicios preferidos. El sistema evalúa los gimnasios disponibles y muestra una lista ordenada por puntaje de recomendación.

Caso de uso 2: Mostrar gimnasios disponibles

El usuario puede revisar todos los gimnasios registrados en el sistema, incluyendo comuna, precio mensual, membresías y servicios ofrecidos.

Caso de uso 3: Crear solicitud de reubicación

Después de revisar las recomendaciones, el usuario selecciona un gimnasio y el sistema crea una solicitud de reubicación con estado pendiente.

5. Implementación

La aplicación fue implementada en Python utilizando Tkinter para la interfaz gráfica. La lógica principal se encuentra separada en archivos dentro de la carpeta src. El archivo models.py contiene las clases del sistema, sistema.py contiene la lógica de recomendación y gui.py contiene la interfaz gráfica.

La base de datos inicial se encuentra cargada directamente en el código como datos de demostración. La versión actual considera 26 gimnasios o sedes de Santiago y alrededores. Los precios y horarios se consideran referenciales, ya que en la realidad pueden variar por promociones, matrícula, contrato y sede.

7. Dificultades encontradas

Una dificultad del proyecto fue definir un criterio simple para recomendar gimnasios. Se resolvió utilizando un sistema de puntaje que considera comuna, presupuesto, tipo de membresía y servicios solicitados. Otra dificultad fue separar la interfaz gráfica de la lógica del sistema, lo cual se solucionó organizando el proyecto en distintos archivos.

8. Bugs presentes

Actualmente no se detectan bugs críticos. Una mejora futura sería permitir guardar usuarios, gimnasios y solicitudes en archivos externos para conservar los datos después de cerrar la aplicación.
